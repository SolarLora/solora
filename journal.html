problems + fixes
Bug — npm permissions
EACCES on global install
System npm tried to write to /usr/local/lib which requires root. Using sudo works but creates files owned by root causing future issues. Fixed by redirecting global prefix to home directory — one time setup, works forever.
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
Bug — heredoc corruption
Bash commands ended up inside the files
Pasting multi-line heredoc commands from chat into terminal caused the shell to write the cat command itself into the file as line 1. Both wrangler.toml and index.js were corrupted. Fix: generate files as downloadable artifacts and copy them with cp. No shell interpretation, no corruption.
Cloudflare
Worker had no edit button
The Worker was created with static assets which disables the browser code editor entirely. Cloudflare AI confirmed the issue and recommended Wrangler CLI. Ended up being the right solution — local deploy pipeline is more reliable than browser editing anyway.
Arduino — Linux
dialout group for USB serial
Arduino IDE on Linux needs the user in the dialout group to access USB serial ports. Runs instantly but requires logout/login to take effect. Without it the device appears in the port list but uploads fail silently.
sudo usermod -a -G dialout $USER

reflection
Honest summary
About 40% engineering, 60% fighting tooling. Both are real work. Switching OS on a deployment day was ambitious but the Linux environment is genuinely better for this project long term. The heredoc lesson — download files, don't paste commands — will not need to be learned again. API is live, site is up, tomorrow starts with a working environment.
Key insight
Generate config files as downloadable artifacts and copy them into place. For any file with complex content — wrangler.toml, index.js, shell scripts — this is more reliable than any amount of careful heredoc quoting. The extra step of downloading takes 10 seconds. Debugging a corrupted TOML file takes considerably longer.

Bug — Website
Fake stats showing on solora.network
The catch block in index.html was serving hardcoded values (3 devices, 1.2 MH/s) when the API was unavailable — meaning the site always showed fake data. Fixed: replaced with honest dashes. Added "coming soon" note that auto-hides when real data loads.
honesty fix resolved
Bug — Firmware
Peer count inconsistent on devices
HB_INTERVAL was 60s and peer timeout was 120s — dead devices stayed counted for up to 3 minutes after going offline. Fixed: HB_INTERVAL reduced to 20s, peer timeout tightened to 45s in both activePeers() and networkHashRate().
timing bug resolved
generateMeshId() — snprintf compile fix
// Before — compiler rejected split arg
snprintf(buf, sizeof(buf), "%04X",
  (uint16_t)(esp_random() & 0xFFFF));

// After — variable extracted first
uint16_t randVal =
  (uint16_t)(esp_random() & 0xFFFF);
snprintf(buf, sizeof(buf),
  "%04X", randVal);
Peer timeout fix
// Before
#define HB_INTERVAL 60000UL
lastSeen < 120000

// After
#define HB_INTERVAL 20000UL
lastSeen < 45000

reflection
Was this a productive day?
Yes — maintenance sessions are essential. Fake website stats undermine trust even with placeholder numbers. Unreliable peer counts make the product feel broken. Fixing infrastructure before adding features is the right order of operations. Small fixes, real improvement.
Key lesson
The peer timeout problem was compounding — 60s heartbeats plus 120s timeout meant a 3 minute window of stale data. Tightening both together to 20s and 45s gives a much more accurate picture. When fixing timing bugs always look at both the send interval and the receive timeout together, not in isolation.

Bug — Deduplication
All heartbeats dropped after first packet
Sequence-number approach used buf[3] as the sequence ID. In HeartbeatPacket buf[3] is the role field — always 0, 1, or 2. Every heartbeat from the same device had the same sequence so all were dropped. Rewrote as a 25s time window per origin+type. Fixed immediately.
root cause found resolved mesh
Bug — Struct Size
Silent mesh failure after removing field
Removed localHour from HeartbeatPacket when time sync was stripped. This changed struct size from 25 to 24 bytes. The len >= sizeof() check on receivers failed silently — all heartbeats dropped. Fixed: replaced field with uint8_t reserved. Never remove packed struct fields.
struct alignment resolved

key decisions
Naming
Solar + LoRa = Solora
SolarBit taken by BitAxe. ROAM taken by Solana/Samsung. Solora: clear, unique, globally pronounceable. Sounds like hardware not a coin.
Power
WiFi off saves 100mA
Workers call WiFi.mode(WIFI_OFF) after role assignment. Single biggest power saving — more than CPU frequency and LoRa power combined. Essential for solar.
Mining
Owner's BTC address only
Each device uses owner's own address. Pool pays them directly. We never touch funds. Avoids money transmission liability entirely.
"The device works for its owner, not for its maker. No server dependency. No account. Works forever if the company disappears."

reflection
Biggest insight
The product philosophy emerged organically from a series of individual technical and legal decisions. It wasn't planned. By the end of the day every decision made independently pointed back to the same principle. That consistency is what makes the product coherent rather than a collection of features.
Key lesson learned
When mesh communication breaks after a firmware change — check struct sizes first. The HeartbeatPacket byte count change was invisible at compile time but broke everything at runtime. Going forward: static_assert(sizeof(HeartbeatPacket) == 25) added as a compile-time guard.
